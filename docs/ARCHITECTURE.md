# BoxSofa 正式架构方案

## 目标

当前线上站点是静态原型，适合展示和收集反馈。下一阶段升级为真实运营版本：

- 前台：商品浏览、分类、详情、购物车、注册登录、提交订单、订单查询、在线客服。
- 后台：老板和客服两个角色，支持商品上架、SKU、库存、订单、物流单号、客户、客服聊天。
- 数据库：Supabase PostgreSQL。
- 图片视频：优先 Supabase Storage；如果视频较大或流量增长，再迁移到 Cloudflare R2。
- 部署：Next.js 可部署到 Vercel；如果继续 Cloudflare 生态，也可以部署到 Cloudflare Pages。
- 支付：先不接真实支付，订单状态由商家确认；以后接 Stripe。

## 推荐模块

1. 商品中心
   - 款式：单人沙发、双人沙发、三人沙发、沙发组合。
   - SKU：颜色、组合、价格、库存、主图 3-5 张。
   - 款式详情：每个款式 1 张详情长图，1 个款式视频。

2. 订单中心
   - 客户提交订单后进入 `pending_confirm`。
   - 客服或老板联系客户确认付款方式。
   - 确认收款后改为 `paid`，再进入发货流程。
   - 后续 Stripe 接入后，Stripe webhook 自动把订单改为 `paid`。

3. 会员系统
   - 客户累计已确认付款金额满 EUR 300 后成为会员。
   - 会员下一次购物享受 9 折。
   - 只有订单状态为 `paid`、`shipped`、`completed` 时才累计消费。

4. 仓库和物流
   - 当前为单仓发货。
   - 商品库存按 SKU 管理。
   - 物流后台手动录入物流公司、运单号和查询链接。
   - 前台展示跨境物流预估 23-30 天。

5. 客服聊天
   - 前台所有页面悬浮在线客服。
   - 后台客服聊天页查看会话并回复。
   - 初期可用 Supabase Realtime，后期可升级为 WhatsApp、邮件或客服系统集成。

## 权限

- 老板：商品、库存、订单、客户、客服聊天全部可管理。
- 客服：订单查询、物流录入、客户维护、客服聊天；默认不允许改价格和删除商品。
- 客户：只能查看自己的订单、地址、聊天记录。

## 支付预留

现在不启用 Stripe。数据库已经保留：

- `orders.payment_provider`
- `orders.payment_reference`
- `orders.paid_at`

以后接 Stripe 时新增：

- 创建 Checkout Session 的 API。
- Stripe webhook 接收支付成功事件。
- webhook 校验后更新订单为 `paid`。
- 自动累计客户消费金额并更新会员状态。
